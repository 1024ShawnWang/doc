
#1.回忆：git保存的是 一系列 快照(blob对象）

#2. 提交对象包含一个快照 和 一个指向快照的指针

#4. 首次提交对象没有父对象
    其他普通提交会有父对象
    多个分支合并，会有多个父对象
    （ 类比 链表，以及多个链表的合并，链表的值就是快照，链表的.pre就指向父对象 )
     ( 对比，链表向后指；但是这个向前指

#5. 分支指向 提交对象

分支的创建
1. git branching testing #我们了解到分支的本质，那么创建分支 只是创建了 一个可移动的指针


2. git如何知道目前在哪个分支上呢？ head指针会告诉git ,因为head指针又 指向分支
2.1 上述的创建分支命令并不会移动head的指向，所以你仍然位于创建分支前的分支上

3. git log --onelin --decorate  来查看各个分支所指的对象（肯定是提交对象)


分支的切换
1. git checkout testing #切换到testing分支

2. git checkout -b <newbranchname>来做到创建，并立刻切到分支
