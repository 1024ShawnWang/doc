
1. 撤销提交: git commit --amend
1.1 它会将暂存区的东西提交到 最近的版本，（所以你需要将漏掉的东西，添加到暂存区)
1.2 如果暂存区没有东西（commit已经清空了暂存区），这条命令会认为 你是想修改 提交说明，会启动文本编辑器，编辑的信息会覆盖 刚刚commit的 提交说明

2. 撤销 进入暂存区 的文件: git reset <have_added_file> 要小心的命令？为什么

3. 撤销 对文件的修改的标记: git checkout <have_modified_file> , 
3.1 因为 **文件修改的标记是否存在** 由系统负责的,  文件修改标记<==>文件真的修改了，所以撤销标记就必须将文件也还原

4. git中，任何 已提交(commit) 的东西，总是可以 恢复 的
